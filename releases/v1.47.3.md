# v1.47.3 — Lift allowed_dirs restriction

**Released:** 2026-04-18
**Type:** Simplification

`validate_path` no longer enforces `ALLOWED_DIRS` containment — any path resolves. The `dirs` CLI argument is now a CWD hint for relative-path resolution, not a security barrier. The `FS_MCP_FLAG` escape hatch survives (still skips interactive review in `propose_and_review`), but path access bypass is the new default. Tests updated; no upstream API changes.
