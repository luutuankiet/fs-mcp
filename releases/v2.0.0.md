# v2.0.0 — Go rewrite, single-step tools, auto-bootstrap, portal detect

**Breaking, major bump.** fs-mcp is now a single static Go binary. The Python package is frozen at the v1.x line.

## Why v2

v1 was a 3077-line Python `server.py` with 23 tools, a Streamlit UI, fakeredis, duckdb, google-genai, and pyfiglet in its dep tree. Cold-start was around 500 ms – 1 s. Tool list cost ~23 k tokens per registration. Deps (jq, yq, rg, rtk) were the user's problem to install — with the common failure of `apt install yq` silently installing the wrong yq.

v2 is a single binary. ~11 MB. Cold-start ~10 ms. `tools/list` under 4 k tokens. First run self-installs the CLI deps at pinned versions.

## The 8-tool core (`read_files`, `grep`, `jq`, `yq`, `run_command`, `directory_tree`, `edit`, `create_directory`)

Dropped from v1: `analyze_gsd_work_log`, `read_media_file`, `list_allowed_directories`, `search_files`, `get_file_info`, `list_directory_with_sizes`, `list_directory`, `append_text`, `move_file`, `write_file`, `grounding_search` (placeholder), the Streamlit `web_ui`, and the two-step `propose_and_review` / `commit_review` pair.

Renames: `query_jq` → `jq`, `query_yq` → `yq`, `grep_content` → `grep`. So the tool names don't collide with other MCP servers behind a proxy.

`edit` is a native-style single-step find-replace (Claude Code parity): `{file_path, old_string, new_string, replace_all?}`. Fails on 0 matches or ambiguous match. The v1 `propose_and_review` → `commit_review` session round-trip was pure friction for an agent and didn't survive review.

`write` was collapsed into `edit` via sentinels — `old_string=""` creates a new file (errors if it exists), `"OVERWRITE_FILE"` replaces the whole file, `"APPEND_TO_FILE"` appends. This surfaces accidental full-file rewrites as explicit `OVERWRITE_FILE` intent in agent transcripts (the sessionless server can't replicate Claude Code's "must Read first" guard, so we lean on intent-naming instead).

## Auto-bootstrap

First start of fs-mcp (or `fs-mcp --doctor`) downloads and installs to `$HOME/.local/bin`:

- `jq` 1.8.1 (jqlang/jq)
- `yq` v4.52.5 (**mikefarah**/yq — not the Python one)
- `rg` 14.1.1 (BurntSushi/ripgrep)
- `rtk` 0.37.0 (rtk-ai/rtk, optional, soft-fail)

Resolution order: managed (`~/.local/bin/<tool>`) → user's PATH → install to managed. Existing copies on PATH are accepted only if they match the pinned version exactly; otherwise fs-mcp installs its own alongside. Shell rc files are never modified — `$PATH` is prepended inside the server process only.

`FS_MCP_PREFER_SYSTEM=1` flips the preference toward user-installed.

## Portal root

v1 required `fs-mcp /path/to/dir`. v2 picks a default:

```
$FS_MCP_ROOT  >  CLI arg  >  headless heuristic  >  $HOME  >  /
```

**Headless**: `$SSH_CONNECTION` set, `$XDG_SESSION_TYPE=tty`, or both `$DISPLAY` and `$WAYLAND_DISPLAY` unset. Startup logs the rule that fired. `fs-mcp --print-root` shows the decision without launching.

## Ripgrep symlink/NAS safety

v1 would OOM-crash when a client mounted the server at `/` on a NAS with symlink loops (parallel ripgrep walkers blew up the work queue). v2's `grep` always passes `--no-follow --one-file-system --max-filesize=50M --threads=2`. When the root resolves to `/` or the path is on nfs/cifs/fuse.sshfs, `--max-depth=4` is forced. Verified against a symlink-cycle fixture — ripgrep returns in ms and the server stays up.

## Hardening pass — what landed since the v2 scaffold

- **`_meta` decoration on every CallToolResult.** Two keys:
  - `anthropic/maxResultSizeChars: 500000` — asks Claude clients to allow 500K-char payloads before truncating, so big reads don't get clipped client-side.
  - `fs-mcp/portal_root: "<root>"` — tells the agent what directory their relative paths resolve against. Lives in `_meta` so it stays out of the response body the agent reads as actual data.
- **Image passthrough restored.** `read_files` returns proper MCP `ImageContent` blocks for `png/jpg/jpeg/gif/webp/bmp/ico/tiff` up to 5 MB, alongside the structured JSON metadata. SVG stays text. Mirrors v1.47.0 ImageContent semantics that were dropped during the rewrite.
- **`run_command` schema cleaned.** The `raw: bool` flag is gone — file-write redirects (`>`, `>>`, `&>`, `2>`, `| tee`) are the escape hatch. Description still names rtk + the redirect bypass so agents self-discover. Removing the knob keeps the compression contract enforced by default; agents can't accidentally default-true their way out of it.
- **CRLF preservation in `edit`.** v1 silently rewrote CRLF files as LF (latent data-corruption bug discovered during the v1↔v2 regression sweep). v2 keeps `\r\n` intact while still matching LF patterns. The auto-recovery uses CRLF-aware substitution before falling back to Unicode-confusable normalization.
- **Auto-update on cold start.** `internal/updater/` checks GitHub Releases on each launch (24-hour cache, 1.5 s timeout, atomic rename + `syscall.Exec` on swap). Replicates the `uvx fs-mcp@latest` UX without per-host SSH — restart your MCP client and you're current. Pin via `install.sh v2.0.3` (writes a marker that freezes auto-update until you re-install without a version). `FS_MCP_NO_AUTO_UPDATE=1` or `--skip-update` opt out.
- **Makefile stamps `git describe` into `main.version`** so local builds participate in the upgrade ladder instead of staying `dev`.

## Structured content, no stringified JSON

Every tool returns a native JSON object in MCP `structuredContent`. v1 wrapped everything in `TextContent(text=json.dumps(x))`, which surfaces in clients as nested escaped strings. v2 uses Go struct → `structuredContent` directly.

Agents see:

```json
{"count":2,"results":[{"id":1,"active":true},{"id":3,"active":true}],"timed_out":false}
```

Not:

```json
"{\"count\":2,\"results\":\"[{\\\"id\\\":1,...}]\"}"
```

## No sandbox

`run_command` has no allowlist. Path scope is not enforced. The portal metaphor is "you have SSH to this box" — the agent's calls are the operator's calls. If that's wrong for your threat model, don't expose fs-mcp to that agent.

The `FS_MCP_FLAG` dangerous-mode sentinel is gone, because there is nothing left to gate.

## Dropped surface (all of this no longer exists)

- Streamlit web UI (port 8123). Use your MCP client.
- HTTP server on 8124 by default. HTTP is now opt-in via `--http <addr>`; stdio is the default.
- Gemini schema-compat transforms. TBD as a follow-up if Gemini breakage is observed in the field — v2 leans on the MCP SDK's native schema output.
- GSD-Lite log analysis and auto-dump. Out-of-scope for a portal.
- The `fakeredis`, `duckdb`, `google-genai`, `pyfiglet`, `streamlit`, `tiktoken` deps. Gone with Python.
- Image relay (`IMAGE_RELAY_URL`), `read_media_file`. Replaced by native MCP `ImageContent` passthrough in `read_files`.
- Per-tool hard caps (100-line truncation on jq/yq results, 15 k char `MATCH_TEXT_MAX_LENGTH`, 50 MB `MAX_FILE_SIZE`, etc.). Bounded by machine memory only.

## Install

```bash
curl -fsSL https://raw.githubusercontent.com/luutuankiet/fs-mcp/v2/scripts/install.sh | sh
# or: download tarball from GitHub Releases and extract fs-mcp to ~/.local/bin
```

Platforms: `linux-amd64`, `linux-arm64`, `darwin-amd64`, `darwin-arm64`. Windows is not supported.

## Upgrade from v1

**The Python `uvx fs-mcp` install path is deprecated.** v2 ships only as a single Go binary via GitHub Releases.

If your MCP client is configured as `{"command": "uvx", "args": ["fs-mcp", ...]}`, replace with `{"command": "fs-mcp"}` after running the install script. Subsequent restarts auto-update — no per-host action needed once v2 is in.

PyPI v1 is left intact at `v1.47.3` for anyone pinned to that line; no `1.48.0` deprecation banner ships. The deprecation lives in this release note and in this repo's README — uvx-based clients aren't actively broken, they just stop receiving fixes.
