# v2.0.1 — `cwd` hint reaches the agent

Patch on top of v2.0.0. One semantic fix.

## What changed

The `_meta.fs-mcp/portal_root` hint added in v2.0.0 was invisible to the model. MCP `_meta` is protocol-level metadata that the client consumes (Claude Desktop / Code use it for things like `anthropic/maxResultSizeChars`) but it's stripped before the tool result reaches the LLM. So an agent calling `read_files("foo.txt")` had no in-band signal about what `foo.txt` was being resolved against.

v2.0.1 moves the hint into the response body where the model actually reads it:

- `structuredContent.cwd: "/home/ken"` — top-level key in the structured payload.
- `content[0].text` — the auto-mirrored JSON now includes the `cwd` key too, so clients that surface `content[]` to the model (rather than `structuredContent`) get the same signal.

`_meta.anthropic/maxResultSizeChars` stays where it is — that one is genuinely client-side.

## Before / after

**v2.0.0:**
```json
{
  "_meta": {
    "anthropic/maxResultSizeChars": 500000,
    "fs-mcp/portal_root": "/home/ken"   // model never sees this
  },
  "content": [{"type":"text","text":"{\"created\":false,\"path\":\"/tmp/x\"}"}],
  "structuredContent": {"created":false,"path":"/tmp/x"}
}
```

**v2.0.1:**
```json
{
  "_meta": {"anthropic/maxResultSizeChars": 500000},
  "content": [{"type":"text","text":"{\"created\":false,\"cwd\":\"/home/ken\",\"path\":\"/tmp/x\"}"}],
  "structuredContent": {"created":false,"cwd":"/home/ken","path":"/tmp/x"}
}
```

## Upgrade

Restart the MCP server. Auto-updater fetches v2.0.1 on cold start.

No tool contract changes; `cwd` is purely additive. None of the existing tool output structs had a `cwd` field at the root, so no collision risk.
