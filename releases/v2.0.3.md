# v2.0.3 — GSD Reader auto-dump returns

Patch on top of v2.0.2. Restores the v1 behavior of pushing `gsd-lite/`
markdown artifacts to a GSD-Reader server on every edit, dropped during
the v2 Go rewrite.

## What changed

When `edit` writes `WORK.md`, `PROJECT.md`, or `ARCHITECTURE.md` inside a
directory literally named `gsd-lite/`, the tool now schedules a
fire-and-forget `POST` to `${GSD_READER_REMOTE}/upload-markdown/<last-two-path-segments>`
with a JSON body `{work, project, architecture, base_path}`. Multiple
rapid edits to the same gsd-lite/ dir coalesce into one upload (10s
debounce). Zero overhead — and zero network calls — when
`GSD_READER_REMOTE` is unset.

## Config

| Env var | Purpose |
|---|---|
| `GSD_READER_REMOTE` | Base URL of the reader server. Required to enable. |
| `GSD_READER_USER` | Basic-auth username (optional). |
| `GSD_READER_PASS` | Basic-auth password (optional, paired with USER). |

## Files changed

- `internal/tools/gsddump.go` (new) — debounced uploader + artifact detector
- `internal/tools/edit.go` — one-line hook after the atomic write
