# v2.0.8 — per-dep skip env + longer download timeout

Two small fixes surfaced during the v2.0.7 fleet rollout.

## Why

**Per-dep skip** — a fleet host with GLIBC 2.35 (Ubuntu 22.04 arm64) couldn't run upstream rtk binaries (require GLIBC 2.39). The blunt fix was `--skip-bootstrap`, which disables ALL dep auto-fetch on that host — jq/yq/rg/duckdb stop updating too. Too coarse.

**Longer download timeout** — a slow residential mac connection tripped the 30s self-update download budget (`auto-update: swap failed: write tmp: context deadline exceeded`). The github.com download endpoint isn't always fast.

## What changed

**`FS_MCP_SKIP_DEPS` env var.** Comma-separated dep names. Listed deps are left entirely alone — no upstream API call, no download, no swap. Doctor reports them as `skipped (FS_MCP_SKIP_DEPS)` with whatever `--version` reports for the on-disk binary. Symlink your own binary into `~/.local/bin/<name>` and the skip leaves it intact.

```
$ FS_MCP_SKIP_DEPS=rtk fs-mcp --doctor
DEP      VERSION    SOURCE     PATH
...
rtk      rtk 0.29.0 skipped (FS_MCP_SKIP_DEPS) /root/.local/bin/rtk
```

**Download timeout 30s → 90s** in `internal/updater/swap.go`. Same budget shape, just 3× the headroom for slow connections.

## Configuration

| Name | Default | Meaning |
|---|---|---|
| `FS_MCP_SKIP_DEPS` | unset | Comma-separated dep names (`rtk`, `jq`, `yq`, `rg`, `duckdb`) to skip entirely. Skipped deps stay at whatever's on disk. Useful when an upstream binary won't run on this host (glibc mismatch, missing CPU features, etc.) and you've staged a working alternative manually. |

## Upgrade

Standard auto-update path. With the v2.0.5 cache-fallback updater, a restart is all it takes — no per-host SSH required.

**Migration tip for hosts pinned via `--skip-bootstrap`.** If a host went onto `--skip-bootstrap` in v2.0.7 just to dodge one bad dep, switch to per-dep skip:

```
# was: proxy_admin patch host config:{args:["--skip-bootstrap"]}
# now: proxy_admin patch host config:{args:[], env:{FS_MCP_SKIP_DEPS:"rtk"}}
```

The host gets back auto-upgrades for the rest of its deps.

## Files changed

- `internal/bootstrap/install.go` — `isSkippedDep(name)` helper reads `FS_MCP_SKIP_DEPS`; `ensureOne()` early-returns a `skipped` Status before any fetch/download work.
- `internal/updater/swap.go` — `downloadTimeout` constant 30s → 90s.
