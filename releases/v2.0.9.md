# v2.0.9 — grep & tree skip machine-generated dirs by default

`grep` and `directory_tree` now filter out universally noisy directories (`node_modules`, `.git`, `dist`, `build`, `target`, `.venv`, `__pycache__`, `.next`, `.terraform`, etc.) by basename, anywhere in the walk. Cuts token waste on any repo where the agent was previously paging through vendored code.

Opt out per-call with `include_ignored: true` when you actually need to traverse vendored packages. Borderline names (`bin`, `lib`, `env`, `.github`) are intentionally NOT filtered.

**Full Changelog**: https://github.com/luutuankiet/fs-mcp/compare/v2.0.8...v2.0.9
