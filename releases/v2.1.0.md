# v2.1.0 — `hq`: query HTML with CSS selectors, no full-doc dump

New tool **`hq`** brings jq/yq-style querying to HTML. Point it at a file, hand it a CSS selector, and get back just the slice you want — text, attributes, tables, or structured JSON — instead of paging an entire noisy document into the agent's context.

**Orientation-first.** Call `hq` with no selector and you get `outline` mode: a structure-only tree (tag, id, classes, child-count, text-length per node) of `<body>`. See the shape of a 500 KB page in a few hundred tokens, then drill in with a selector. Script/style/svg/head/comments are stripped by default.

**Modes:** `outline` (structure, no text payload), `text`, `attr` (single name, `*` for all, or a comma-separated list), `html` (inner), `outer`, `json` (`{tag,id,classes,attrs,text,n_children}`), `count`, `table` (`{headers,rows}`). A `limit` (default 50) keeps results bounded and sets a `truncated` flag when exceeded.

**Escape hatch.** `mode=raw` returns matched outer HTML with no stripping and no whitespace normalization; `strip: []` disables noise removal entirely; `attr` accepts `*` or a comma list. When an opinionated mode boxes you in, `run_command` (curl/sed/etc.) remains the universal fallback.

**In-process, zero deps added.** Unlike `jq`/`yq`/`query_duckdb` (which shell out to bootstrapped binaries), `hq` compiles `github.com/PuerkitoBio/goquery` directly into the CGO-free static binary. No per-host download, no arm64-musl/GLIBC/Rosetta gaps — every host gets `hq` on the next self-update with nothing to install.
